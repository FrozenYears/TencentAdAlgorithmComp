# 2026 腾讯广告算法大赛 - 往届经验研究

## 一、2025年比赛概况

### 1.1 赛题信息

| 项目 | 详情 |
|------|------|
| 赛题名称 | 全模态生成式推荐（All-Modality Generative Recommendation） |
| 参赛规模 | 全球8400+选手，2800+支队伍 |
| 数据集 | TencentGR-1M（百万级）和TencentGR-10M（千万级） |
| 冠军队伍 | Echoch团队（华中科技大学、北京大学、中国科学技术大学） |
| 冠军奖金 | 200万元人民币 |

### 1.2 数据特点

**数据分布**:
- 曝光行为: 94.63%
- 点击行为: 2.85%
- 转化行为: 2.52%

**关键挑战**:
- 94%的曝光噪声淹没仅2.5%的高价值转化信号
- 需要从海量无序的用户行为样本中提取有效信息

### 1.3 评估指标

**主要指标**:
- HitRate@10: 命中率
- NDCG@10: 归一化折损累积增益

**权重设计**:
- 复赛阶段引入行为类型权重
- 转化行为权重为2.5倍

## 二、冠军方案深度解析

### 2.1 Echoch团队方案

**核心创新**: 三级会话体系与逐位置行为条件化建模

#### 技术亮点

**1. 逐位置行为条件化（Per-position Action Conditioning）**
- 通过FiLM特征空间变换层实现点击/转化预测的差异化表征
- 三机制协同：
  - FiLM特征调制
  - Gated Fusion门控融合
  - Attention Biasing注意力偏置

**2. 模型架构**
- 引入HSTU（Hierarchical Sequential Transduction Unit）架构
- 结合RoPE位置编码
- 单卡显存占用减少5GB

**3. 语义ID方案**
- 提出RQ-KMeans语义ID
- 用多模态大模型生成统一Embedding
- 使用残差量化K-Means离散化
- 引入Random-k正则化，防止模型忽略精细层级信息

**4. 优化策略**
- Muon优化器（用于稠密权重）与AdamW优化器（用于稀疏Embedding）混合使用
- 显存占用锐减45%，收敛速度提升40%

**最终成绩**: 线上核心指标+0.013提升

#### 特征工程详解

**时间特征**:
- 构建不同时间粒度（秒、周、月）的特征
- 区分随机点击和真实兴趣

**会话特征**:
- 将用户连续行为切片为Request、Session、Visit Session三个层级
- 模拟用户浏览和决策心理

**多模态特征**:
- 利用文本、图像等embedding
- 通过多模态大模型生成统一Embedding

### 2.2 leejt团队方案（亚军）

**核心思路**: 大数据，大模型，处理数据脏乱差问题

#### 技术亮点

**1. 数据处理**
- 共享词表+哈希编码：巧妙处理超大规模数据
- 对高维连续特征进行降维去噪
- 将连续向量离散为可控的离散表示

**2. 特征工程**
- 对缺失率高、线下验证效果不佳的模态特征直接舍弃
- Session划分+异构时序图
- 引入图结构刻画用户、广告与语义特征之间的关系

**3. 模型架构**
- 采用G-MLP/SASRec混合架构处理多模态噪声
- 处理数据质量问题

**4. 工程优化**
- 多环节显存优化
- 训练、梯度检查点、Dataloader并行
- GPU利用率拉到0.8

### 2.3 技术创新奖方案

**团队**: 料峭春风吹酒醒（中国科学院计算技术研究所）

**核心创新**: 统一生成式检索与排序的单模型范式

#### 技术亮点

**1. 模型设计**
- Decoder-only生成式模型
- 联合建模用户下一个感兴趣的item和用户对item的行为
- 统一训练目标：语义ID生成损失 + 动作预测损失

**2. 先进技术组件**
- FlashAttention
- SwiGLU前馈网络
- RMSNorm
- RoPE位置编码
- DeepSeek-V3风格的混合专家（MoE）架构

**3. 语义ID构建**
- 专用decoder-only Transformer + InfoNCE损失提取协作嵌入
- 碰撞解决机制：自动搜索下一个最近的聚类中心

**4. 特征工程**
- 原始稀疏用户/物品属性
- 多模态物品表示
- 跨多个时间窗口的物品流行度统计
- 离散和连续时间特征

**5. 训练推理优化**
- 混合精度训练
- 分离稀疏/稠密优化器更新
- 分组GEMM + KV缓存加速

## 三、2019年比赛经验

### 3.1 赛题信息

| 项目 | 详情 |
|------|------|
| 赛题名称 | 移动App广告转化率预估 |
| 任务目标 | 预测App广告点击后被激活的概率 |
| 评估指标 | LogLoss |
| 冠军队伍 | nju_newbie |

### 3.2 冠军方案特点

**数据清洗**:
- 构建train_dev_df、train_df、dev_df、test_df
- 通过train_dev_df和dev_df进行有监督的模型训练

**特征提取**:
- 人群定向特征
- 投放时段特征
- 多值特征的主副键
- 历史统计特征
- 聚合统计特征
- 五折统计特征
- Word2Vec和DeepWalk embedding特征

**模型组合**:
- CIN（xDeepFM）
- Key-Value记忆网络
- Word2Vec
- DeepWalk
- 四种模型单元分别提取特征，concat输入到MLP

**关键技巧**:
- 掩盖5%的广告，保证训练集中也能出现无曝光的广告
- 解决新广告冷启动问题

### 3.3 2019年经验总结

**特征工程**:
- 从历史角度和全局角度构建特征
- 前一天、最近一天、历史所有、前n天和五折交叉统计
- 基础特征扩展为商品id和账户id等

**模型选择**:
- 树模型（LightGBM、XGBoost）是基础
- 深度模型（xDeepFM、DIN等）是进阶
- 模型融合是提升关键

## 四、2020年比赛经验

### 4.1 赛题信息

| 项目 | 详情 |
|------|------|
| 赛题名称 | 广告受众基础属性预估 |
| 任务目标 | 预测用户的年龄和性别 |
| 评估指标 | Accuracy（年龄+性别） |

### 4.2 亚军方案特点

**特征工程**:
- 对所有的creative进行编码
- 目标编码：构建标签预测解，由模型实现平滑
- Kfold和去一法

**模型创新**:
- BERT模型应用
- Focus/target bert：只将target相关信息注入
- 标签预测阶段，学习率降低50倍

**关键启示**:
- 前期用户点击行为较少，按用户基础属性推荐
- 后期用户有点击行为后，按行为推荐
- 稀疏实体富信息，容易形成信息孤岛

### 4.3 2020年经验总结

**数据处理**:
- 降低数据穿越带来的影响
- 尝试贝叶斯平滑等方法

**模型训练**:
- FFM模型训练时要注意过拟合
- 一定要划分验证集

**内存优化**:
- 数据规模较大时，减少对Numpy、pandas的使用
- 用list等原生数据结构减少内存依赖

## 五、踩坑点与注意事项

### 5.1 数据相关问题

**数据穿越**:
- 问题：历史数据泄露导致线下评估虚高
- 解决方案：使用贝叶斯平滑等方法降低影响

**数据质量**:
- 问题：多模态特征缺失率高、噪声大
- 解决方案：特征筛选和聚合，直接舍弃低质量特征

**数据分布**:
- 问题：行为类型分布极度不均衡
- 解决方案：使用行为类型权重，强调高价值转化事件

### 5.2 模型训练问题

**过拟合**:
- 问题：FFM等模型容易过拟合
- 解决方案：一定要划分验证集，关闭auto-stop

**内存优化**:
- 问题：大规模数据导致内存不足
- 解决方案：减少对Numpy、pandas的使用，用list等原生数据结构

**显存优化**:
- 问题：大规模模型显存不足
- 解决方案：使用Muon优化器，显存占用锐减45%

### 5.3 工程实现问题

**代码共享**:
- 注意：比赛主张自主创新，注意code share问题
- 建议：多探索fancy一点的方法，不要拘泥于某一块的实现/trick

**特征处理**:
- FFM特征格式：field+feature_index+value的格式转换是关键
- 数值特征处理：参考Kaggle开源例子，或输出boosting树模型的叶子节点值来离散

**推理一致性**:
- 注意：推理和训练保持一致，特别是温度系数等超参数

### 5.4 提交相关问题

**提交格式**:
- 文件名：submission.csv
- 编码：UTF-8（无BOM）
- 格式：每行"样本ID, 预估日曝光（或pCVR）"，逗号分隔，无空格，无表头

**提交限制**:
- 每天限提交3次结果

## 七、工程优化技巧

### 7.1 显存优化

**Muon优化器**:
- 通过Newton-Schulz迭代做正交化
- 显存占用锐减45%
- 二阶动量的显存开销显著降低

**混合精度训练**:
- 兼容torch.cuda.amp
- 实现内存高效训练
- 在现代GPU上显著减少显存占用

### 7.2 训练加速

**收敛速度**:
- Muon优化器收敛速度提升40%
- 各方向更新步长更均匀，收敛更快、最终效果更好

**计算优化**:
- 分组GEMM：优化矩阵计算
- 静态计算图：提升训练效率

### 7.3 推理优化

**向量检索**:
- 使用Faiss构建ANN索引
- 在线用用户向量在Faiss索引中检索Top-K广告候选

**推理解耦**:
- 用户侧：在线实时运行完整的Transformer
- 广告侧：离线预计算广告向量，构建Faiss ANN索引

### 7.4 数据处理优化

**高效数据加载**:
- 使用文件偏移量进行随机访问
- 减少内存依赖，使用原生数据结构

**特征处理**:
- 左填充到固定长度，从后往前保留信息
- 多种特征类型的padding和缺失值填充

## 九、总结与建议

### 9.1 技术趋势

- **生成式推荐**: 从判别式到生成式的范式转变
- **多模态融合**: 文本、图像、视频等多元数据的整合利用
